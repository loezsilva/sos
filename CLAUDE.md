# Buzz — Instruções para agentes

## Idioma
Responda **sempre em português do Brasil**. Todo código gerado (variáveis, funções, classes, comentários, mensagens, docstrings) deve estar em português.

## Projeto
SaaS de atenção imediata para círculos íntimos. Ver `openspec/config.yaml` e `openspec/specs/buzz/spec.md` para requisitos completos.

## Referências obrigatórias
- Design system: `docs/DESIGN.md`
- HTMLs de inspiração: `docs/sos/*/code.html`
- Specs OpenSpec: `openspec/specs/`

## Django
Fat Models, Thin Views. CBVs + ModelForms. ORM para bulk updates. Código mínimo.

## OpenSpec
Use `/opsx:propose` para novas mudanças, `/opsx:apply` para implementar, `/opsx:archive` para finalizar.
